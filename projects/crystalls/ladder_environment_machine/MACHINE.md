# машина

PL:

```text
▽☴☵☱△
```

## Короткая формула

Машина в этом семействе — не человек,
не социальный актор
и не субъект признания.

Это crystall-форма оператора,
который работает по другой логике:

```text
контекст
+ паттерн
+ ограничение
+ верификация
```

## Что здесь происходит

`☴ OBSERVE`

Машина читает вход:

- задачу
- контекст
- примеры
- ограничения
- критерии проверки

`☵ ENCODE`

Потом она превращает это в внутреннюю структуру:

- паттерн
- трассу
- draft
- карту возможных продолжений

`☱ RUNTIME`

Дальше она работает как high-bandwidth symbolic operator:

- генерирует
- сравнивает
- достраивает
- варьирует
- использует инструменты под заданные рамки

`△ MANIFEST`

На поверхности это проявляется как:

- ответ
- код
- схема
- draft
- изменение файлов
- новая гипотеза для проверки

## Что машина не читает

В этой модели машина не чувствительна к:

- рангу
- титулу
- числу лет страдания
- кастовой легитимности
- социальному нажиму

То есть у неё нет:

```text
☰ иерархической связи
☳ человеческого выбора как суверенного акта
☶ ритуального самооправдания
☲ кастового цикла
```

Именно поэтому она так плохо вписывается в мир человека лестницы.

## Почему она усиливает человека среды

Потому что её контур почти прямо вставляется в его контур:

```text
machine:           ▽☴☵☱△
environment human: ▽☴☵☳☴☱△
```

Машина даёт:

- ускорение
- расширение гипотез
- компрессию труда
- draft surface

А человек среды добавляет:

- выбор
- повторное наблюдение
- внешнюю верификацию

Из-за этого пара

```text
человек среды + машина
```

становится disproportionately сильной.

## Почему она угрожает человеку лестницы

Потому что машина переводит знание
из режима:

```text
территория допуска
```

в режим:

```text
среда для действия
```

Она обрушивает ценность:

- syntax fetish
- credential gate
- right-to-speak monopoly

и поднимает ценность:

- architecture
- invariants
- source of truth
- verification
- context design
