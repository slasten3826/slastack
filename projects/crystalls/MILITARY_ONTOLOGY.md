## 2. Reading

```text
FLOW
```

Есть поток различий, сил, тел, желаний, угроз, ресурсов и возможностей.

```text
CONNECT
```

Различия связываются в отношения.
Появляется поле соотнесения:

- своё / чужое
- сильное / слабое
- разрешённое / запрещённое
- верх / низ
- порядок / угроза

```text
ENCODE
```

Отношения кодируются как признаки, роли, стороны, лагеря, статусы и типы риска.

```text
CHOOSE
```

Кодировка начинает требовать выбора.

Субъекту уже недостаточно просто быть.
Он должен занять позицию:

- с нами или против нас
- лоялен или подозрителен
- свой или чужой
- правильный или опасный

```text
LOGIC
```

Выбор получает оправдание.

Появляется доктрина:

- мораль
- порядок
- долг
- истина как сторона
- необходимость защиты
- право на принуждение

```text
CYCLE
```

Доктрина повторяется.
Повторение закрепляет кодировку.

Цикл может быть:

- ритуальным
- дисциплинарным
- бюрократическим
- образовательным
- пропагандистским
- бытовым

```text
RUNTIME
```

После достаточного числа циклов режим становится средой.

Субъект уже не просто верит в карту.
Он действует внутри неё как внутри рабочего мира.

```text
MANIFEST
```

Военная онтология проявляется как конкретная форма:

- война
- армия
- фронт
- мобилизация
- режим лояльности
- институт подозрения
- социальная машина стороны

## 3. Missing Operators

В этой трассе нет:

```text
☴ OBSERVE
☷ DISSOLVE
```

Это важно.

Без `OBSERVE` военная онтология не видит себя как режим.

Без `DISSOLVE` она не растворяет собственную разметку.

Поэтому она склонна к hardening:

```text
различие -> сторона -> доктрина -> повторение -> среда
```

Короткая формула:

```text
военная онтология = blind encoding cycle without dissolve
```

---

## 4. Relation To Taiji Table

Этот документ не описывает `taiji table` целиком.

Военная онтология — не root.
Она не является всем различием и не является всей table-сборкой.

Она является одним жёстким crystallization path:

```text
различие становится стороной
сторона становится выбором
выбор получает доктрину
доктрина циклически закрепляется
цикл становится runtime
runtime проявляет социальную машину
```

То есть:

```text
taiji table != военная онтология
военная онтология = hard path from taiji table
```

## 5. Difference From Game

Игра может расти из похожего table-напряжения, но не обязана идти этой трассой.

В game trace обычно появляются:

```text
OBSERVE
возврат к FLOW
повторный CHOOSE
безопасный CYCLE
```

Военная онтология в этой форме идёт иначе:

```text
CONNECT -> ENCODE -> CHOOSE -> LOGIC -> CYCLE -> RUNTIME
```

Она быстрее превращает различие в сторону и доктрину.
Игра чаще возвращает игрока к наблюдению и новому потоку.

## 6. Difference From Chakras

Reference:

```text
военная онтология:
▽☰☵☳☶☲☱△

чакры:
▽☰☵☳☶☱☲△
```

Delta:

```text
☲☱ -> ☱☲
```

Военная онтология:

```text
LOGIC -> CYCLE -> RUNTIME
```

Чакры:

```text
LOGIC -> RUNTIME -> CYCLE
```

Военная онтология сначала прогоняет доктрину через повторение и только потом делает её средой.

Чакры сначала превращают карту в внутренний runtime, а потом стабилизируют её циклами практики и самочтения.

## 7. Short Formula

Военная онтология — это crystallization path, в котором различие кодируется как сторона, сторона требует выбора, выбор получает доктрину, доктрина циклически закрепляется, а закреплённый цикл становится runtime.

Её опасность не в наличии различия.

Её опасность в том, что она не содержит собственного `OBSERVE` и `DISSOLVE`, поэтому склонна принимать свою разметку за весь мир.
