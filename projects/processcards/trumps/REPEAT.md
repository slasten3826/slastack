# REPEAT

## 0. Status

Working trump draft.

This is not final balance text.
This is not locked rules text.

Legacy breadcrumb:

```text
XVII Star
```

Flavor text:

```text
What entered once will enter again.
```

Core edge:

```text
☱ -> ☲
RUNTIME -> CYCLE
```

---

## 1. Core Identity

`REPEAT` is a chain-deepening trump.

It does not create a fresh independent law.
It does not invent a new effect.
It repeats the effect of the trump that caused it.

If no trump caused it,
it has no effect.

Short formula:

```text
repeat the parent trump effect
or do nothing
```

---

## 2. Reading

`☱ RUNTIME` here is not passive installation.

It means the already-entered law is still held in machine reality.
The parent trump's force has not yet dissipated.

`☲ CYCLE` here is not ordinary hand filtering.

It means the entered trump-effect passes once more through the machine.
A second echo of the same disturbance.

So this trump reads as:

```text
what has already entered the chain
enters again
```

This is not:

```text
copy
```

And not:

```text
recall of the distant past
```

It is:

```text
living echo of the immediate cause
```

---

## 3. Core Law

When `REPEAT` becomes known:

- if it was caused by a trump effect, repeat the effect of the trump that caused it
- if it was not caused by a trump effect, `REPEAT` has no effect

Short formula:

```text
parent trump exists -> repeat parent effect
no parent trump -> no effect
```

---

## 4. Parent Trump Reading

`REPEAT` should not look for some vague previous event.

It cares only about the direct trump that caused `REPEAT` to resolve.

That trump is the:

```text
parent trump
```

So the logic is:

1. trump `A` resolves
2. `A` causes `REPEAT`
3. `REPEAT` resolves
4. `REPEAT` repeats effect `A`

This keeps the card causal, local, and readable.

---

## 5. Ecology Consequence

`REPEAT` does not bypass trump ecology.

If repeating the parent trump effect causes:

- more trump exposure
- more chain growth
- trump-zone pressure
- chamber release / shuffle-back
- changed deck state

all of that happens normally.

So `REPEAT` is not a fake replay in a vacuum.

It repeats the parent effect
inside the already changed machine.

This is important.

---

## 6. Why This Works

`REPEAT` is strong only in the right context.

If it appears outside a trump-caused chain,
it is empty.

If it appears inside a trump-caused chain,
it can make the chain much deeper and stranger.

That makes it:

- contextual
- dangerous
- highly nonlinear
- specifically tied to trump turbulence
- not a generic all-purpose power card
- less direct than `FOOL`, but more amplifying

---

## 7. Relation to Trump Ecology

`REPEAT` does not bypass trump ecology.

It uses:

- ordinary parent-trump cause tracking
- ordinary ordered resolution
- ordinary trump queue
- ordinary chain close handling
- ordinary trump-zone law after the parent effect repeats

Its specificity is not in escaping the machine.
Its specificity is in running one more cycle
inside the already-in-motion machine.

---

## 8. Restrictions

`REPEAT` does not:

- copy ordinary minor effects by default
- repeat some vague last remembered event
- ignore trump ecology
- suppress chamber release / shuffle-back
- create a second unrelated effect choice

It repeats only the direct causing trump effect.

---

## 9. Design Character

`REPEAT` should feel:

- blunt
- recursive
- chain-deepening
- more like continuation than explosion
- dangerous when found inside living trump machinery
- empty outside that machinery
- less archetypal than `FOOL`
- less structural than `RECAST`
- more like an echo than a command

This is intentional.
