## Main Public Repositories

### `processlang`

<https://github.com/slasten3826/processlang>

Read as:

- operator/process language layer
- deeper formal core
- ontology and invariant work

### `daofetch`

<https://github.com/slasten3826/daofetch>

Read as:

- terminal-facing artifact
- symbolic frontend
- near-product manifestation of deeper ideas

### `planGOD`

<https://github.com/slasten3826/planGOD>

Read as:

- live architecture field
- raw stack laboratory
- long-arc direction traces

### `packetcli2machine`

<https://github.com/slasten3826/packetcli2machine>

Read as:

- machine-first repository
- runtime interaction slice
- observability / build / execution path for another machine

### `X112`

<https://github.com/slasten3826/X112>

Read as:

- process-graphics runtime prototype
- Linux display / X11 compatibility spine
- test branch for applying stack ontology to a real systems problem
- proof branch that the stack can accelerate architecture -> docs -> code -> tests

Important:

- `X112` is not the final `X12`
- it should be read as an intermediate but real branch
- it exists to test whether the stack survives contact with graphics/runtime reality

### `packet-slop`

<https://github.com/slasten3826/packet-slop>

Read as:

- packet-neural machine branch
- transfer of `packet` logic onto neural-compute rails
- working `L1-L3` machine body field
- live bridge between packet ontology and neural execution

Important:

- this is not a polished ML repository
- this is not a transformer-fork project
- this is a machine-first body for `packet-neural` research and execution
